Matchbox是什么(what’s Matchbox)

Matchbox 是一种服务，它将机器与profile文件匹配起来，通过PXE启动、部署容器Linux集群。Profile可以指定kernel/initrd,内核参数,iPXE配置, GRUB配置,容器Linux配置 ,Cloud-Config或者其它配置。Matchbox为客户端提供经身份验证的gRPC接口,比如terraform，它可以通过二进制文件，RPM包，容器镜像或者kubernetes安装。

Tutorials(教程)

Getting started(入门)

此教程将介绍如何使用Terraform与Matchbox配置Container Linux machines。
你将安装Matchbox服务，配置PXE网络启动环境，然后使用Terraform 配置描述你的基础结构并使用Terraform命令行在Matchbox创建定义的资源。

Matchbox

在指定的server或者Kubernetes Cluster安装matchbox服务。生成TLS证书并按照指示启用gRPC API.把ca.crt,client.crt 和 clinet.key 保存在你本机(e.g. ~/.matchbox).

  • 安装到Cantainer Linux/其它发行系统安装
  • 安装到kubernets
  • 在rkt/docker运行

验证matchbox HTTP终端服务是否可用.

验证用户证书和key是否可以访问gRPC API.

Terraform

安装v0.9+版本的Terraform.

安装terraform-provider-matchbox 插件

添加上述插件到~/.terraformrc

First Cluster

Clone matchbox 源并查看Terrafrom样例

让我们从simple-install开始。通过simple-install，任何从matchbox PXE启动的机器将会把Contanier Linux安装到dev/sda,重启并完成SSH key设置。它不是一个集群，但我们稍后后谈到。

通过创建terraform.tfvars文件来配置variables.tf中的变量。

现在Terraform可以与matchbox服务通讯、创建资源了。

让我们回顾下Terraform的配置文件，在了解下mathbox。

Provider

对裸机资源来熟matchbox被配置一个provider平台。

Profiles

机器配置指定内核，initrd，内核参数，Container Linux配置，Cloud-config或者其他用于网络启动、配置裸机的配置文件。这个配置文件将使用当前稳定版本Container Linux内核和initrd通过PXE启动机器并提供Container Linux配置文件，这个配置文件指定应执行磁盘安装和重启操作。了解更多Container Linux 配置

Groups

匹配器组基于像MAC，UUID等标签的机器与机器特定值中的不同配置文件和模版进行匹配。这个组中没有selector块，所以任何从matchbox网络启动的机器将会匹配到这个组，并使用coreos-install配置文件配置。机器与最匹配的组匹配。

Apply

使用Terraform 配置。

如果你感兴趣：Mathbox为计算机提供配置并维护查询参数

  • iPXE default - /ipxe
  • Ignition default - /ignition
  • Ignition post-install - /ignition?os=installed
  • GRUB default - /grub

Network

Matchbox可以与许多预置的网络配置集成。它不试图成为网络的DHCP server，TFTP server或者DNS server。相反，Matchbox提供iPXE脚本和GRUB配置，这些iPXE脚本和GRUB配置作为配置网络引导计算机的接入点。通过链式加载 iPXE框架来支持 PXE客户端。
在最简单的情况下，支持iPXE的网络可以链接到matchbox，

阅读 network-setup.md获取完整的选项范围。网络管理员有最大灵活性：

  • 可以使用现有的DHCP，TFTP和DNS服务
  • 可以配置子网，架构或者指定计算机委派给Matchbox
  • 可以把matchbox放到一个菜单项之后(timeout and default to matchbox)

如果你之前从未没配置过支持PXE的网络或者你试着配置测试环境，检出 quay.io/coreos/dnsmasq 容器镜像 copy-paste examples并查阅proxy-DHCP部分

Boot

是时候通过网络启动你的计算机了。使用BMC远程管理功能，把启动设备设置为PXE并启动每台机器。

每台机器应该链式加载iPXE，委托给Matchbox，接收其iPXE配置(或者其他支持的配置)并开始配置过程。simple-install示例假定你的机器配置为首先从硬盘启动，仅在请求时启动PXE，但针对不同情况，你可以自定义配置文件。
一旦Container Linux安装完成并重启后，你可以SSH登录，

基于其他目重新配置机器，运行terraform apply,PXE会再次引导它。

Going Further

如果以HA方式部署，Matchbox可以在一个或多个预设站点部署多个节点Container Linux集群。可以通过MAC地址，UUID，区域或者你选择的其他标签来单独匹配机器。通过在内建HTTP assets服务器缓存镜像可以更快地安装。
Container Linux配置可以用于分区磁盘和文件系统，编写systemd单元，网络配置或普通文件，而且可以创建用户。Container Linux节点可以根据需求配置到系统中。
查看创建3节点etcd集群或者3节点kubernetes集群样例。
